1)
- 휴머노이드(humanoid) : 인간의 형태를 모습으로 한 로봇
- '인공지능=로봇' 이란 공식은 항상 성립하지 않음
- 인공지능이란 하나의 소프트웨어 프로그램 일종임
- 인공지능이 저절로 혼자 똑똑해질 수는 없음
- 오늘날 인공지능의 90% 이상은 지도학습 (Supervised Learning) 방식으로 훈련됨
- 또한, 사람이 인공지능의 구조를 설계하고 다양한 최적화 기법을 적용하는 실험을 해야만 더 나은 성능 가능
- 인공지능이 감정을 가질 일은 없음. 다만, '감정을 가지는 척' 하도록 프로그래밍을할 수 있음
- 인공지능의 학습 및 추론이란 데이터를 벡터나 매트릭스 형태의 텐서(tensor)로 만들어 복잡 다양한 행렬 연산일 뿐임
- AI : 지능적인 행동의 자동화
- 약 인공지능 연구에 있어서는 눈부신 발전이 이뤄져 왔으나 강 인공지능에 대해서는 이렇다 할 연구가 진행되지 못함
- 인간의 명령이 없어도 스스로 판단하고 결정을 내리기도함
- 매우 범용성을 가지는 인공지능으로, 범 인공지능이라고도 부름
- 우리의 일상생활에서 만나는 모든 인공지능
- 제한된 환경에서 구체적인 특정 업무를 수행하는 데 있어서 사람과 비슷한, 또는 사람 이상의 성능을 내는 것
- 범용성과 전문성의 구별이 절대적이진 않음
- 넓은 의미의 AI: 자동화 할 수 있는 프로그램 전부
- 좁은 의미의 AI: 요즘 말하는 인공지능. 딥러닝(인공신경망)을 기반으로, 데이터만 가지고 인간의 사고/행동을 표방할 수 있음
- 인공지능 : 사람이 해야할 일을 기계가 대신할 수 있는 모든 자동화에 해당
- 머신러닝 : 명시적으로 규칙을 프로그래밍하지 않고 데이터로부터 의사결정을 위한 패턴을 기계가 스스로 학습
- 인공신경망 기반의 모델로, 비정형 데이터로 부터 특징 추출 및 판단까지 기계가 한번에 수행
- 특징 추출 : 문제 해결을 위해 어떤 정보가 유용할지, 중요한 것을 꺼내는 과정
- 판단 방식 : 추출한 중요 특징으로 부터 인식을 하거나 판단을 모델로 만드는 단계
- 데이터/현상 -> 특징(feature) -> 인식 및 예측
- 특징 추출-사람, 판단-사람 : 규칙 기반 프로그래밍 AI(넓은 의미의 AI). 문제해결에 필요한 주요 정보와 판단 규칙을 사람이 설계. 예) 계산기처럼 사람의 연산 방식을 구현하는 규칙기반프로그래밍
- 특징 추출-사람, 판단-머신 : 머신러닝 기반의 AI. 문제해결에 필요한 주 특징만 사람이 결정. 판단 모델은 수학적 모델링과 기계학습을 통해 구현. 예)전통적인 Machine Learning (회귀분석, 의사결정트리 등)
- 특징 추출-머신, 판단-머신 : 딥러닝 기반의 AI (좁은 의미의 AI). Raw Data로 부터 수학적 모델링과 인공신경망 학습을 통해 모든 것을 구현하는 방식. 예) 이미지/텍스트 데이터를 그대로 처리하는 Deep Learning 기법(CNN, RNN 등)
- 딥러닝만 스마트한 인공지능이고, 규칙 기반 프로그래밍이 쓸모 없는 기술이란 말은 아님. 구현 편의성, 복잡도, 적용 상황에 따라 각 방식의 장단점이 있음

2)
- 딥러닝은 기계가 데이터를 입력받은 후 데이터의 주요한 특징을 알아서 추출하고 이를 바탕으로 의사결정을 하는 기술
- 머신러닝이 다루는 정형 데이터(structured data)는 안정성은 높으나 체계적으로 구조가 고정되어 있어서 유연하지 않은 데이터, 머신러닝 특화
- 머신러닝 기반의 기술이란? 빅데이터 분석 기법. R, SAS, SPSS 과 같은 통계분석 툴
- 실수값 예측 (선형/로지스틱 회귀분석), 카테고리 분류 (의사 결정 트리), 시계열 예측 (ARMA, ARIMA 모형)
- 딥러닝이 다루는 비정형 데이터(unstructure data)
